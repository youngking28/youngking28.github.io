Deploy webapp with idea and tomcat

Posted on 2018-05-23

idea如何结合tomcat，部署web程序。

下载和安装tomcat，这里就不赘述了。

先在idea中配置tomcat。
选择添加tomcat server

设置server名字和home地址


之后， 在idea的module setting(右键工程，选中open module setting)中，先对你的工程设置好相对应的文件夹类型，idea中有Sources（源文件夹），Tests（测试文件夹）， Resources（资源文件夹），TestResource（测试资源文件夹），Exclude（排除文件夹）五种。

其中，Sources中的java文件，会被编译生成class文件，输出到指定path下（在module中可以设置）。
Resources则是项目所需要的文件配置，将会被一起打包放入输出path下，strutcs.xml等框架配置文件必须存放，否则将不会被打包， eclipse则是会一起打包。

设置完成后，去Facets中，创建刚刚moudle下的web子包，其中设置好web.xml的路径， 以及web页面的文件夹路径（WebRoot或者web）。

下一步去Artifacts中，将上面所设置的web包打包成一个 web application expload， 其中需要在WEB-INF下的classes指定上面module的complie output， 这样module编译后的class将会被打包进这个artifacts中。

在这一步中，有时候可能会碰到，资源文件无法被编译（笔者尝试多种方案后，暂时没有找到原因和解决方案），这时可以在artifacts这一步中，添加玩module的complie output，同时将资源文件的文件夹添加进这个artifacts， 就可以了。

最后，在tomcat， deploy选项中，加入这个artifacts， 启动tomcat， 就完成了部署！
